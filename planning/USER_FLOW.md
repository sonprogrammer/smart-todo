# User Flow

## 1. 로그인

- MVP에서는 로그인하지 않는다.
- 사용자는 앱에 진입하면 바로 Todo 조회 흐름으로 이동한다.

## 2. Todo 조회

1. 사용자가 앱에 진입한다.
2. 시스템은 저장된 Todo 목록을 불러온다.
3. 시스템은 각 Todo의 기본 정보를 표시한다.
   - 작업명
   - 설명
   - 우선순위
   - 마감일
   - 하위 작업
   - 완료 상태
4. 사용자는 목록에서 Todo를 확인한다.

### 예외

- 저장된 Todo가 없으면 빈 목록 상태를 표시한다.
- 조회 실패 시 오류를 표시하고 기존 Todo CRUD 흐름은 유지한다.

## 3. Todo 생성

1. 사용자가 Todo 생성을 시작한다.
2. 사용자가 Todo 정보를 입력한다.
   - 작업명
   - 설명
   - 우선순위
   - 마감일
   - 하위 작업
3. 사용자가 저장을 요청한다.
4. 시스템은 입력값을 검증한다.
5. 시스템은 Todo를 저장한다.
6. 시스템은 Todo 목록을 갱신한다.

### 예외

- 필수 입력값이 없거나 형식이 올바르지 않으면 저장하지 않고 오류를 표시한다.

## 4. Todo 수정

1. 사용자가 Todo 목록에서 수정할 Todo를 선택한다.
2. 시스템은 선택한 Todo의 현재 정보를 불러온다.
3. 사용자가 Todo 정보를 수정한다.
   - 작업명
   - 설명
   - 우선순위
   - 마감일
   - 하위 작업
   - 완료 상태
4. 사용자가 저장을 요청한다.
5. 시스템은 수정값을 검증한다.
6. 시스템은 Todo를 업데이트한다.
7. 시스템은 Todo 목록을 갱신한다.

### 예외

- 수정값이 올바르지 않으면 업데이트하지 않고 오류를 표시한다.
- 대상 Todo가 없으면 오류를 표시하고 목록을 다시 조회한다.

## 5. Todo 삭제

1. 사용자가 Todo 목록에서 삭제할 Todo를 선택한다.
2. 사용자가 삭제를 요청한다.
3. 시스템은 대상 Todo를 삭제한다.
4. 시스템은 Todo 목록을 갱신한다.

### 예외

- 대상 Todo가 없으면 오류를 표시하고 목록을 다시 조회한다.

## 6. AI 추천

1. 사용자가 자연어로 할 일을 입력한다.
2. 사용자가 AI 추천을 요청한다.
3. 시스템은 AI 요청의 trace id를 생성한다.
4. 시스템은 자연어 입력을 AI 오케스트레이션에 전달한다.
5. 시스템은 에이전트별 입력, 출력, 오류를 로그로 기록한다.
6. 시스템은 AI 결과를 Todo draft로 생성한다.
   - 작업명
   - 설명
   - 우선순위
   - 마감일
   - 하위 작업
7. 시스템은 draft와 저장된 Todo를 분리해서 유지한다.
8. 사용자가 draft를 검토한다.
9. 사용자가 필요한 경우 draft를 수정한다.
10. 사용자가 draft 저장을 승인한다.
11. 시스템은 승인된 draft를 실제 Todo로 저장한다.
12. 시스템은 Todo 목록을 갱신한다.

### 예외

- AI 처리에 실패하면 trace id와 오류 로그를 남긴다.
- AI 출력 검증에 실패하면 draft를 저장하지 않는다.
- AI 실패 또는 검증 실패 시 사용자는 수동 Todo 생성 흐름으로 이동할 수 있다.
- 사용자가 승인하지 않은 draft는 실제 Todo로 저장하지 않는다.
