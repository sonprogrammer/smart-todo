# Frontend

## FE-01
- ID: FE-01
- 목표: Todo 목록 조회 화면과 빈 상태를 구현한다.
- 담당: Frontend
- 선행 작업: 없음
- 완료 조건: 저장된 todo 목록을 조회해 화면에 표시하고, 데이터가 없을 때 빈 상태가 표시된다.

## FE-02
- ID: FE-02
- 목표: 수동 todo 생성 폼을 구현한다.
- 담당: Frontend
- 선행 작업: FE-01
- 완료 조건: 사용자가 직접 입력한 todo가 저장되고 목록에 반영된다.

## FE-03
- ID: FE-03
- 목표: Todo 수정, 완료 처리, 삭제 UI를 구현한다.
- 담당: Frontend
- 선행 작업: FE-02
- 완료 조건: 사용자가 todo를 수정, 완료 처리, 삭제할 수 있고 변경 결과가 목록에 반영된다.

## FE-04
- ID: FE-04
- 목표: 자연어 입력 화면과 draft 생성 요청 UI를 구현한다.
- 담당: Frontend
- 선행 작업: FE-03
- 완료 조건: 사용자가 자연어를 입력해 draft 생성을 요청하고 생성 결과를 확인할 수 있다.

## FE-05
- ID: FE-05
- 목표: Draft 검토, 수정, 승인 저장 UI를 구현한다.
- 담당: Frontend
- 선행 작업: FE-04
- 완료 조건: 사용자가 draft를 수정하거나 승인해 실제 todo로 저장할 수 있다.

## FE-06
- ID: FE-06
- 목표: AI 실패 fallback UI와 trace log 확인 UI를 구현한다.
- 담당: Frontend
- 선행 작업: FE-05
- 완료 조건: AI 처리 실패 시 수동 작성 흐름으로 이동할 수 있고 trace id와 에이전트 로그를 확인할 수 있다.

# Backend

## BE-01
- ID: BE-01
- 목표: Todo 데이터 모델과 저장소 초기 구조를 구현한다.
- 담당: Backend
- 선행 작업: 없음
- 완료 조건: todo의 제목, 설명, 우선순위, 마감일, 하위 작업, 완료 상태를 저장하고 조회할 수 있다.

## BE-02
- ID: BE-02
- 목표: Todo 생성과 목록 조회 API를 구현한다.
- 담당: Backend
- 선행 작업: BE-01
- 완료 조건: 수동 입력 todo를 생성할 수 있고 저장된 todo 목록을 조회할 수 있다.

## BE-03
- ID: BE-03
- 목표: Todo 수정, 완료 처리, 삭제 API를 구현한다.
- 담당: Backend
- 선행 작업: BE-02
- 완료 조건: 기존 todo를 수정, 완료 처리, 삭제할 수 있다.

## BE-04
- ID: BE-04
- 목표: Draft 데이터 모델과 todo 저장 분리 구조를 구현한다.
- 담당: Backend
- 선행 작업: BE-01
- 완료 조건: draft가 사용자 승인 전 실제 todo 저장소에 저장되지 않는다.

## BE-05
- ID: BE-05
- 목표: 자연어 입력을 todo draft로 변환하는 AI 오케스트레이션을 구현한다.
- 담당: Backend
- 선행 작업: BE-04
- 완료 조건: 자연어 입력 요청마다 trace id가 생성되고 구조화된 draft가 반환된다.

## BE-06
- ID: BE-06
- 목표: Draft 수정과 승인 저장 API를 구현한다.
- 담당: Backend
- 선행 작업: BE-02, BE-04
- 완료 조건: 수정된 draft를 승인하면 실제 todo로 저장되고 draft와 todo가 구분된다.

## BE-07
- ID: BE-07
- 목표: AI 출력 검증과 실패 처리 흐름을 구현한다.
- 담당: Backend
- 선행 작업: BE-05
- 완료 조건: AI 출력 검증 실패 시 오류 상태를 반환하고 기존 todo CRUD가 계속 동작한다.

## BE-08
- ID: BE-08
- 목표: 에이전트 실행 로그 조회 API를 구현한다.
- 담당: Backend
- 선행 작업: BE-05, BE-07
- 완료 조건: trace id 기준으로 에이전트별 입력, 출력, 오류, 최종 결과 로그를 조회할 수 있다.
